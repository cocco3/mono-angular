# Theme

Color variables that are defined based on their intended use or meaning. They
provide context and convey information, such as status, state, or action.

All variables are sourced from [Untitled UI](https://www.untitledui.com/components/color-styles).

Themes support a light mode and dark mode. The color scheme will match your system settings by default, or can be manually applied by setting the `color-scheme` property.

- For **light-mode**, apply the `color-scheme: light` property
- For **dark-mode**, apply the `color-scheme: dark` property

<div>
  {Object.entries(defaultTheme.colors).map(([groupKey, group]) => (
    <div key={groupKey} style={{ marginBlock: 40 }}>
      <Heading>{colorDictionary[groupKey].name}</Heading>

      <div class="list sb-unstyled">
        {Object.entries(group.values).map(([weight, hex]) => {
          const variableName = `${group.id}-${weight}`
          const cssVariable=`var(--${variableName})`
          const lightSchemeValue = hex[0]
          const darkSchemeValue = hex[1]

          return (
            <div class="row">
              <div class="variableName">
                {variableName}
              </div>

              <div class="colorChip">
                <div class="color"
                  style={{ backgroundColor: cssVariable}} />
                <div>
                  <div class="variable">{colorNameLookup(lightSchemeValue)}</div>
                  <div class="hex">{lightSchemeValue}</div>
                </div>
              </div>

              <div class="colorChip colorChip-dark">
                <div class="color"
                  style={{ backgroundColor: cssVariable}} />
                <div>
                  <div class="variable">{colorNameLookup(darkSchemeValue)}</div>
                  <div class="hex">{darkSchemeValue}</div>
                </div>
              </div>
            </div>
          );
        })}
      </div>
    </div>

))}

</div>

## Usage

```css
.someClass {
  --_border-color: var(--border-primary);
  border: 1px solid var(--_border-color);
}
.someClass.error {
  --_border-color: var(--border-error);
}
.someClass.warning {
  --_border-color: var(--border-warning);
}
.someClass.dark {
  /* apply the dark color scheme */
  color-scheme: dark;
}

/* you can also create global helper classes for dark/light modes */
.light-mode {
  color-scheme: light;
}
.dark-mode {
  color-scheme: dark;
}
```

## Usage

Use the CSS `font` property and combine the weight with the variable:

```css
/* bold and large */
font: 700 var(--type-display-lg);

/* regular and small */
font: 400 var(--type-text-sm);
```

# UiSlotDirective

Use this directive with optional [content placeholders](https://angular.dev/guide/components/content-projection#multiple-content-placeholders).

## Example

### app-child-component.ts

In this example, the child component template has 2 optional content placeholders called `start` and `end` that should be wrapped in a div only if the content has been provided.

In the child component, get all the `UiSlotDirective` content children, and check if they are available with the `useSlots` function.

```ts
import { Component, contentChildren, input } from '@angular/core';
import { UiSlotDirective, useSlots } from '@cocco3/angular-ui';

@Component({
  selector: 'app-child-component',
  template: `
    @if (hasSlots().start) {
      <div class="icon">
        <ng-content select="[uiSlot='start']" />
      </div>
    }

    @if (!!label()) {
      <div class="label">
        {{ label() }}
      </div>
    }

    @if (hasSlots().end) {
      <div class="icon">
        <ng-content select="[uiSlot='end']" />
      </div>
    }
  `,
})
export class AppChildComponent {
  label = input<string>();

  private slots = contentChildren(UiSlotDirective);
  protected hasSlots = useSlots(this.slots, ['start', 'end']);
}
```

### app-parent-component.ts

In the parent component, add the `uiSlot` attribute on the appropriate elements.

```ts
@Component({
  imports: [UiSlotDirective, AppChildComponent],
  selector: 'app-parent-component',
  template: `
    <app-child-component label="My label">
      <span uiSlot="start">Leading slot</span>
      <span uiSlot="end">Trailing slot</span>
    </app-child-component>
  `,
})
export class ParentComponent {}
```
